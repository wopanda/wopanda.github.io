基于Obsidian+Hexo+Github的博客搭建流程

发表于2023-05-10|更新于2023-05-16|生活
|阅读量:

搭建博客是为了什么？

这是我的第一篇博文，其实很早就想搭建一个正式的博客了，可能当时是因为这样能体现一定的技术力。

当然，现在这个原因还是没变，只是多了点其他的感受，更多的是想要将自己日常生活中遇到的问题及其解决方案，还有一些折腾效率工具的经历分享出来。

一方面，想以此推进自己成长，另一方面，也想以此来记录我的生活。

关于搭建自己的博客

为什么是这个方案？

因为我个人是obsidian的爱好者，很多文字都习惯记录于ob，所以通过ob作为文本编辑器发布博客，成为了我的不二选择。

也恰好，因为在摸鱼的时候，看到一篇博文Hexo + Obsidian + Git 完美的博客部署与编辑方案 | EsunR-Blog，吸引了我，也就自然而然选择了hexo+github的这个方案。

事实上，hugo搭建可能性能会更好浅谈我为什么从 HEXO 迁移到 HUGO - 少数派 (sspai.com)，但是对我而言，满足需求即可（偷懒

博客搭建说明

因为网上优秀资源较多，所以不会做重复造轮子的事，更多的是做一个资料的汇总以及坑点说明。

搭建博客需要用到：node.js + Git + obsidian + hexo + github。

体验到的效果，只需要在OB中进行文字编辑，点击按钮即可发布博客，且无需等待多久。

折腾记录

安装Hexo

  • 前提

    • 安装Git
    • 安装Node.js
      • 下载 | Node.js 中文网 (nodejs.cn)
      • 环境变量需要设置，参考视频 Node.js安装及环境变量配置_哔哩哔哩_bilibili
      • 版本要大于12.3，这个与视频不同

  • OB目标目录下，调用命名

    • 安装hexo npm install -g hexo-cli
    • 初始化：hexo init

使用 github.io 展示网页

  • 测试本地是否可用
    • hexo g
    • hexo s
    • hexo clean
    • 将网址复制到浏览器上打开
  • 发布到pages
    • 【静态博客】部署到 GitHub_哔哩哔哩_bilibili
    • npm install hexo-deployer-git --save
    • hexo d

安装第三方主题及配置信息

  • 方法一：
    • 将下载好的主题解压于 theme 目录下，并重命名
    • 修改 _config.yml 的 theme 信息
  • 方法二（推荐）
    • Butterfly 安裝文檔(一) 快速開始 | Butterfly

使用 Githhub Action 自动化部署

  • ♪(^∇^*)欢迎肥来！使用 Github Action 自动部署 | 安知鱼 (anzhiy.cn)
  • 13.hexo完成github action自动部署_哔哩哔哩_bilibili
git add .
git commit -m "github action update"
git push origin main
  • 下方的写法会更简洁，方便复制运行：
git add . && git commit -m "github action update" && git push origin main

日志自动分类设置

  • Hexo + Obsidian + Git 完美的博客部署与编辑方案 | EsunR-Blog
    • 【Hexo插件系列】日志的自动分类插件 hexo-auto-category | ESON
$ npx husky add .husky/pre-commit "npx hexo generate && git add ."

Obsidian 一键发布

  • 采用obsidian插件shell commands插件进行调用推送git命令，命令如下：
cd 你的博客路径 && git add . && git commit -m "github action update" && git push origin main

ps：shell commands插件不会用的话，可以参考这个视频 教你怎么在Obsidian里用Python，让你的Obsidian更强大！_哔哩哔哩_bilibili

  • 之后，用另一个 Commander 插件，将上述命令设置为按钮，即可体验一键发布。
